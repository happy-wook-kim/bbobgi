# 경마 v2 — 긴 트랙 + 이벤트(돌·부스터) 설계 문서

작성일: 2026-07-13 · 기반: `2026-07-13-horse-race-design.md`

## 1. 개요

경마 트랙을 화면보다 2~3배 길게 만들고 카메라가 선두를 따라가게 한다. 레인에 🪨 돌(밟으면 비틀거리며 정지)과 ⚡ 부스터(먹으면 급가속) 이벤트를 추가한다. **이벤트는 연출 전용** — 도착 시각을 바꾸지 않으므로 꼴찌(`winnerIndex`)는 여전히 `pickWinner`가 확정한다.

## 2. 결정 사항 요약

| 항목 | 결정 |
|---|---|
| 트랙 | 카메라 없음 — 다른 게임들처럼 **트랙 전체(출발 벽~결승선)가 항상 화면 안**. 출발 벽은 화면 왼쪽(12px), 결승선은 오른쪽(100%−46px). max-width 640px |
| 레이스 시간 | 약 12초 — 선두 그룹(마지막 둘 제외)은 6.0~7.5초에 조기 골인, 결승 상대는 ~11.5초+지터, 꼴찌는 250~550ms 뒤 (마지막 둘의 장기 접전) |
| 접전 연출 | 결승 상대는 **꼴찌의 인접 레인**에 배정(연출용, 공정성 불변). 셋째 꼴찌 골인 직전부터 듀오 중심으로 줌인(최대 1.7배, 뷰포트 중앙 클램프 — 가장자리 레인도 잘리지 않음), 결판 후 줌아웃 |
| 이벤트 | 말당 정확히 3개(랜덤 종류), 트랙 진행도 1/4·2/4·3/4 지점에 일정 간격 고정 배치. **절대 시간 고정**(남는 시간은 일반 구간이 흡수) — 돌: 0.35~0.55초 정지, 부스터: 끊기지 않고 1.0~1.2초 유지하며 진행도 18~21%를 내달림 |
| 마커 | 밟기 전엔 전부 ❓(정체 숨김). 밟는 순간 공개 — 🪨는 남고 ⚡는 소모 후 사라짐. 발동 중 말에 💫(돌)/🔥(부스터) 표시 |
| 속도 변화 | 편차 기반 밴드 선택 — 자기 평균선보다 뒤처지면 질주(1.45~1.9배), 앞서면 숨 고르기(0.4~0.7배), 근처면 랜덤. 도착 시각 편차(SPREAD 0.5초)를 좁혀 팩이 붙어 달린다. 판당 평균 선두 교체 ≥4회·최장 독주 ≤45%를 테스트로 보장(부스터 1초 유지가 만드는 정당한 선두 시간 포함) |
| 이름 표시 | 카메라에 밀려 사라지지 않도록 왼쪽 열 대신 **말 머리 위 이름표** |
| 화면 폭 | `.screen.race`를 카드·사다리처럼 full-bleed(100vw)로 |

## 3. 엔진 확장 — `src/engine/race.ts`

```ts
type RaceEventKind = 'rock' | 'boost';
type RaceEvent = { kind: RaceEventKind; t: number; duration: number; x: number };
type RaceProfile = { finishTime: number; waypoints: Waypoint[]; events: RaceEvent[] };
```

waypoints 생성을 구간 기반으로 재구성한다:

1. 이벤트 0~2개의 (종류, 시작 t, duration)을 슬롯 분할로 겹치지 않게 배치 (15%~80% 시간대)
2. 구간 경계 = [0, 이벤트 시작/끝…, finishTime]. 1.6초 넘는 일반 구간은 반으로 나눠 속도 지터(역전 연출)
3. 구간별 진행량 가중치: 일반 = 길이×(0.8~1.2), 돌 = 0.02(거의 정지), 부스터 = 길이×2.5
4. 누적 후 0~1 정규화 → waypoints. 이벤트의 x = 해당 구간 시작 지점의 정규화 진행도

기존 보장 유지: x·t 단조 증가, `f(finishTime)=1`, 꼴찌 최후 도착. **기존 테스트 5개는 수정 없이 통과해야 한다.**

## 4. 컴포넌트 — `HorseRace.tsx`

- rAF 루프는 그대로, 렌더 시 카메라 오프셋 계산: `cam = clamp(선두px − 화면폭×0.55, 0, 월드폭 − 화면폭)`
- 말 구조 변경: `.race-horse`(위치) > `.race-glyph`(🐎, scaleX 반전·달리기/비틀 애니메이션) + `.race-tag`(이름표) + `.race-fx`(💫/🔥)
- 현재 시각이 이벤트 구간 안이면 `is-stumbling`/`is-boosting` 클래스
- 순위 뱃지는 결승선 오른쪽(월드 끝)에 절대 배치, 결승선은 월드 레벨 세로선 하나

## 5. 테스트

- 신규: 이벤트 개수(0~2)·종류·x∈(0,1) / 시간대(15~80%)·비겹침 / 돌 구간 속도 ≪ 평균 & 부스터 구간 속도 > 평균 (progressAt으로 행동 검증)
- 기존 race.test.ts 5개 무수정 통과

## 6. 범위 밖 (YAGNI)
